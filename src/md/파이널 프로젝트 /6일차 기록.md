### 🛠️ 사용 기술

- next.js

### 💡 배운 점

- 데이터 베이스는 기본적으로 영어 또는 숫자로 저장하기 때문에, 데이터의 통일성과 유지보수 관리 측면에서도, 화면에 보여주는 글자와 데이터 키값을 따로 관리할 수 있어 유지 보수 측면에서 더 효율적이고 전문적인 방법이다.

### 😵 어려웠던 점

- 데이터 베이스의 언어와 next.js에서 화면으로 보여주는 언어가 달라서 다르게 적용을 했었는데, 이렇게 되면 표준 위반이여서, 이 부분을 수정하는데 처음하는 부분이라서 수정하는 시간이 좀 오래 걸렸다. 

### 🔧 해결 방법

- 해당 데이터 베이스를 기반으로 한 객체를 새로 만들어서 데이터 베이스에 맞는 한국어 라벨을 만들어주고, 다시 새로운 배열로 만들어서 화면에 한국어로 보여주는 작업을 거쳤다.

### 😊 느낀 점
