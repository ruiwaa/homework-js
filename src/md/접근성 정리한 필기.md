# 🌐 웹 접근성(Accessibility) 핵심 정리 가이드

이 문서는 시각 장애인(스크린 리더 사용자), 키보드 사용자 등 모든 사용자가 웹 사이트를 동등하게 이용할 수 있도록 하기 위한 핵심 원칙들을 정리한 문서입니다.

## 1. 시각적 숨김 처리 (`sr-only`)

디자인상 텍스트를 숨겨야 하지만, 스크린 리더는 읽어야 할 때 사용합니다.

- **원칙:** `display: none`이나 `visibility: hidden`은 스크린 리더도 무시하므로 사용하지 않습니다.
- **해결:** CSS로 요소를 1px 크기로 줄여 눈에만 안 보이게 만드는 `.sr-only` 클래스를 사용합니다.
- **활용:** `label` 텍스트 숨기기, 섹션의 제목(`h2`) 숨기기, 이미지에 대한 추가 설명 등.

## 2. 인터랙티브 요소 (버튼 & 폼)

사용자가 클릭하거나 입력하는 요소에 대한 접근성 설정입니다.

- **Label 연결:** 모든 `input`은 반드시 `label`과 `id`로 1:1 연결되어야 합니다.
- **Button 타입 명시:**
  - `type="submit"`: 폼 데이터를 전송하고 페이지를 이동/새로고침할 때.
  - `type="button"`: 단순 자바스크립트 기능 실행(예: 포인트 조회, 최대 적용 등) 시 사용. (명시하지 않으면 기본값은 `submit`이므로 주의)
- **ARIA 상태 값:**
  - `aria-pressed`: 좋아요, 온/오프 버튼처럼 **상태가 유지되는 토글 버튼**에 사용.
  - `aria-selected`: 탭 메뉴처럼 **여러 개 중 하나가 선택**된 상태를 나타낼 때 사용.
  - **주의:** 일반적인 실행 버튼(결제하기, 취소 등)에는 상태 값을 넣지 않습니다.

## 3. 복잡한 위젯 구조 (Tab 메뉴)

탭 메뉴와 같이 구조적인 연결이 필요한 경우 표준 역할을 부여합니다.

- **구조:** 부모(`role="tablist"`) > 자식(`role="tab"`) > 내용(`role="tabpanel"`) 순서로 구성합니다.
- **연결:**
  - 버튼(`tab`)은 `aria-controls="패널ID"`를 가져야 합니다.
  - 패널(`tabpanel`)은 `aria-labelledby="버튼ID"`를 가져야 합니다.
- **상태:** 현재 활성화된 탭에만 `aria-selected="true"`를 부여하고, 나머지는 `false`로 설정합니다. 비활성 패널은 `hidden` 속성으로 숨깁니다.

## 4. 레이블링 및 대체 텍스트 (`alt` & `aria-label`)

스크린 리더가 사용자에게 들려줄 목소리를 디자인합니다.

- **이미지 `alt`:** "이미지", "포스터" 같은 단어보다는 "이터널 선샤인 영화 포스터"처럼 **구체적인 정보**를 적습니다.
- **`aria-label` 남용 금지:**
  - 이미 버튼 내부에 텍스트가 있다면 `aria-label`은 쓰지 않는 것이 원칙입니다.
  - 라벨 안에 "버튼", "클릭"과 같은 단어를 포함하지 않습니다. (스크린 리더가 이미 "버튼"이라고 읽어주기 때문)
- **장식용 요소:** 의미 없는 아이콘이나 장식용 SVG는 `aria-hidden="true"`를 주어 스크린 리더가 무시하게 합니다.

## 5. 내비게이션 및 실시간 업데이트

사용자의 편의를 돕는 추가적인 장치들입니다.

- **건너뛰기 링크 (Skip Navigation):** 페이지 상단에 본문으로 바로 이동할 수 있는 링크를 제공하여 키보드 사용자의 피로도를 낮춥니다.
- **실시간 알림 (`aria-live`):** 결제 금액 합계처럼 사용자의 동작에 따라 실시간으로 변하는 숫자 정보에는 `aria-live="polite"`를 부여하여 값이 변했음을 목소리로 알려줍니다.
