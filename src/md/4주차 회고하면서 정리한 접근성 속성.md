# aria-label vs aria-labelledby 빠른 정리

## 1. 핵심 판단

**보이는 텍스트 있음? → `aria-labelledby` (권장)**
**보이는 텍스트 없음? → `aria-label`**

---

## 2. aria-label (텍스트 직접 제공)

```html
<button aria-label="메뉴 닫기">✕</button>
<button aria-label="검색">🔍</button>
```

**사용 시기:** 아이콘만 있을 때

---

## 3. aria-labelledby (텍스트 연결) ⭐

```html
<!-- 라벨 -->
<h2 id="title">제목</h2>

<!-- 연결 -->
<div aria-labelledby="title">내용</div>
```

**사용 시기:** 보이는 텍스트가 있을 때

---

## 4. 탭 메뉴 예시

```html
<!-- 탭 버튼 (텍스트 있음) -->
<button id="tab-1" role="tab">카드 관리</button>

<!-- 탭패널 (버튼 텍스트 연결) -->
<div role="tabpanel" aria-labelledby="tab-1">
  콘텐츠
</div>
```

**이유:** 버튼의 "카드 관리"를 탭패널의 이름으로 재사용

---

## 5. tab, tablist, tabpanel

| 역할 | 역할 | 예시 |
|------|------|------|
| **tablist** | 탭 목록 감싸기 | `<nav role="tablist">` |
| **tab** | 클릭 버튼 | `<button role="tab">` |
| **tabpanel** | 탭 콘텐츠 | `<div role="tabpanel">` |

---

## 6. SVG 처리

**순수 장식:**
```html
<svg aria-hidden="true"></svg>
```

**의미 있는 아이콘:**
```html
<svg role="img" aria-label="홈"></svg>
```
