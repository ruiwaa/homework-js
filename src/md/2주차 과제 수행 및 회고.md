```
이렇게 우선적으로 기본 환영메시지 내용의 함수 선언문을 만들었다. 그리고 ai에게 "if문을 사용하지 않고 어떻게 간단하게 만들 수 있을지 힌트를 줄 수 있어?" 라고 물어보았더니, '삼항 연산자'를 이용하여 문제를 해결하라고 알려주었다.

const passwordValidationm = (password) => {

  return !!password
}

또한 OR연산자를 사용하게 되면 8자 이상, 20자 이하 중 하나의 조건에 부합이 안되도 다른 한 쪽의 조건엔 부합이 되서 모두 '참'으로 반환한다.

✅ 해결 방법: 이 부분에 대해서 ai에게 물어보니 새로운 객체 속성을 알려주었습니다.

### 5번 문제) 영화 티켓 가격 계산
### 1단계: 함수 표현식 선언
```
const movieTicketPrice = function (movieType, is_earlybird_discount, audienceCount ){
}
```
### 2단계: 객체로 영화 종류별 가격 관리
```
const movieTypePrice = {
  '일반': 14_000,
  '3D': 17_000,
  'IMAX': 20_000
}
```
💡 객체를 사용하여 영화 종류와 가격을 매칭
💡 나중에 movieTypePrice[movieType]으로 동적으로 가격 조회 가능함

### 3단계: 논리 연산자를 활용하여 조조할인 조건 처리
```
const earlybirdDiscount = (is_earlybird_discount && movieTypePrice[movieType] * 0.2) || 0
```
💡 AND 연산자(&&): 조조 할인이 true일 때만 20% 할인액 반환
💡 OR 연산자(||): 조조 할인이 false이면 0원 적용

### 4단계: 최종 영화 티켓 가격 계산
```
const count = (movieTypePrice[movieType] - earlybirdDiscount) * audienceCount
```
### 5단계: 숫자 타입으로 반환(Number() 함수로 엄격한 타입 변환을 통해 숫자 타입으로 반환)
```
return Number(count)
```

```

## 회고(깨달은 점 및 아쉬운 점)
수업 시간에 선생님이 주시는 문제는 단계별로 생각을 하고 코드를 짤 수 있게 문제 지문이 구성되어져 있어, 문제 지문 순서에 맞게 생각을 깊이하지 않아도 코드를 비교적 쉽게 작성할 수 있었다. 하지만 이번 과제를 풀면서 느낀 점은 크게 3가지가 있는데, 다음과 같다.
<br>
1) 자바스크립트를 이용해서 이런 프로그램?의 로직을 만들어 낼 수 있다.
-> 어떻게 보면 수업 중에 다루는 실습 문제들은 각 챕터별로 관련된 문제들이 나오는데, 이번 과제는 그동안 배웠던 지식들을 활용하여 내가 스스로 코드 로직을 짜보는 시간이기 떄문에 코드를 작성하면서 이런 문제(프로그램)을 실행시킬려면 어떠한 함수의 내용을 작성해야 되는지를 눈과 손으로 조금씩 익혀 볼 수 있는 시간이었다. 
<br>
2) 코드 작성의 한계
-> 1번 문제 풀려고 하는데 도저히 어떻게 풀어야 될지 감이 오질 않았다. 각 조건에 따른 상응하는 값을 반환해야하는데, 그런 개념을 아직까지 수업시간에 배우질 않아서 어떻게 이러한 부분들은 배운 부분을 위주로 만들 수 있을까 고민을 해보다가 결국 ai에게 해당 문제에 코드를 작성하기 위해 알아야 할 개념이 무엇인지를 알려달라고 했다. 처음 듣는 연산자를 언급해줬는데, '삼항 연산자'라는 알게 되었다. 기본적인 연산자 개념을 보고 코드를 짜보았더니, 출력하고 싶은 값들이 나와서 신기했다. 그러나 수업 중에 배운 내용이 아니다보니 알려준 대로 코드를 짰지만 이해가 충분히 되지는 않았다. 

-> 2번 문제부터 5번까지는 1번 문제와 다른 점은 조건들이 2개가 아닌 3개 이상의 조건들이 나오는 문제들이 있었다. 이런 경우에 '삼항 연산자'를 사용하면 오늘 한 개념인데 더 머릿속이 복잡해질 것 같아서, 어떻게 짜야할지 고민이 되고, 또 스스로의 부족함이 느껴지면서 '아... 큰일났다. 앞으로 어떻게 스스로 코드를 짜야하지? '라는 막막함도 들었다. 

-> 막막함만 느끼고 포기하는 것보다 일단 스스로 최대한 시도를 해보자 라고 마음을 고쳐먹고, 스스로 어떻게 풀어야 할지 모르겠으니 ai 에게 이런 문제의 유형을 풀고 싶고, 내가 배운 학습 개념들을 나열해서 보여주며, 내가 배운 개념들을 활용해서 만들 수 있도록 힌트를 줄 수 있냐고 물어보았다.

-> 객체와 논리 연산자 활용 : 조건을 객체로 만들어서 객체의 값을 논리 연산자에 대입하여 조건식? 을 만들었다. 예습을 했을 때 잠깐 if문이라는 것을 들었었는데, 지금 배운 개념이 아니므로, 이제까지 배운 개념들을 최대한 활용해서 문제를 풀고 싶었다. if이 없이 객체와 논리 연산을 활용해서 만드니, 코드가 좀 더 길어진 기분이 들었지만, 그래도 이 두가지를 사용해서 조건에 부합하는 값들을 반환할 수 도 있다는 사실을 알아내서 신기하고 ai 에게 코드를 짜달라고 하지 않고 필요한 개념들을 묻고 전반적인 코드 로직은 스스로 짜는 시간을 가져보려고 노력했다.  

-> 아쉬운 점은 객체와 논리 연산자를 활용하는 것이 2~5번 문제에 다 반영이 되었는데 이게 가장 효율적이고 효과적인 코드 작성인지에 대한 의문도 들고, 좀 더 다양한 방식으로 코드를 작성하지 못한 것이 아쉽게 다가왔다. 
<br>

3) 새로운 자바스크립트 개념과의 연관성
-> 처음에 이런 조건문?은 ai가 if문을 추천해줬는데, 아직 if문에 대한 개념을 접하지 않았기때문에, 쓰지 않았지만 이번 과제가 다음주에 배울 개념들의 예고편? 같은 느낌이 들었다. if문이 없이 함수를 구현하여 원하는 값을 출력하는 법도 배우고, 이후에 이러한 프로그램에 딱 적합한 개념들을 배우게 되면 두 가지 방식의 코드 작성을 할 수 있는 능력의 범위가 확대될 수 있을 것 같아서, 다음주에 배울 새로운 학습 개념에 대한 궁금증이 더 생기게 되었다.
